## 4. Архитектурные проблемы

### 4.1 Магические числа (⚠️ ТРЕБУЕТ ИСПРАВЛЕНИЯ)
| Локация | Значение | Проблема |
|---------|----------|----------|
| converter.py | `MAX_ROWS = 1000000` | Жесткий лимит без объяснения; значение всё ещё продублировано локально в двух методах |
| converter.py | `2000` | Шаг проверки `stop_flag` жестко зашит в коде (встречается ~7 раз) |
| converter.py | `total_rows // 100` | Частота обновления прогресса |

### 4.2 Проблемы с сигналами и связностью (⚠️ ТРЕБУЕТ ПОЛИРОВКИ)
| Локация | Проблема |
|---------|----------|
| gui.py / main.py | Основной перегруз сигнала `conversion_started` устранён. Однако `main.py` (в методе `_start_conversion`) всё ещё напрямую читает состояния виджетов из `gui.py`. **Рекомендуется** передавать параметры через `dataclass`. |

### 4.3 Потенциальные утечки памяти (🔴 ВЫСОКИЙ РИСК)
| Локация | Описание |
|---------|----------|
| converter.py:728-790 | `_write_split_csv` - открытые файлы складываются в словарь `open_files` и остаются открытыми. При огромном количестве уникальных значений программа может превысить лимит ОС на открытые дескрипторы и упасть. |
