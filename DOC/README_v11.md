# TSV_Converter v11.0 - Новая архитектура

## Структура проекта

```
TSV_Converter/
├── main.py              # Точка входа, интеграция GUI и бизнес-логики
├── gui.py               # Модуль интерфейса (PySide6, без QSS)
├── converter.py         # Бизнес-логика (конвертация, утилиты)
├── Converter v.10.6.py  # Старая версия (архив)
└── Документация по ошибкам.md
```

## Ключевые изменения

### 1. Модульная архитектура
- **gui.py** - только интерфейс, никаких бизнес-логики
- **converter.py** - только бизнес-логика, никаких зависимостей от GUI
- **main.py** - связующий слой (Controller в MVC)

### 2. GUI без QSS
Новый интерфейс использует только **нативные возможности PySide6**:
- `QPalette` для цветовых схем
- `QSizePolicy` для адаптивного размещения
- `QLayout` (VBoxLayout, HBoxLayout, GridLayout) для компоновки
- Никаких жёстко заданных размеров (`setGeometry`, `move`)

### 3. Адаптивность
- Автоматическое масштабирование через `apply_scale()`
- Поддержка тем (Светлая/Тёмная) через `StylePalette`
- `QSizePolicy.Expanding` для гибкого изменения размеров

### 4. Устранённые проблемы старой версии

| Проблема v10.6 | Решение v11.0 |
|----------------|---------------|
| Дублирование `get_encoding()` | Единый класс `FileUtilities` |
| Блокировка UI | Потоковая конвертация через `QThread` |
| Жёстко заданные размеры | Только layout-ы + size policies |
| QSS стили в коде | Нативная палитра через `QPalette` |
| Монолитный код 3000 строк | Модульная архитектура (3 файла) |

## Компоненты GUI

### Базовые виджеты
- `StyledButton` - базовый класс кнопок
- `PrimaryButton` - основная кнопка действия
- `SecondaryButton` - второстепенная кнопка
- `DangerButton` - кнопка опасного действия (красная)
- `FileListWidget` - список файлов с Drag & Drop
- `LogTextEdit` - поле логов с авто-прокруткой

### Диалоговые окна
- `SettingsDialog` - настройки приложения
- `ColumnValuesDialog` - выбор значений столбца
- `PivotSettingsDialog` - настройки сводной таблицы
- `TSVPreviewDialog` - предпросмотр файла

### Главное окно
`MainWindow` использует вертикальный `QVBoxLayout` с панелями:
1. Панель файлов (список + кнопки управления)
2. Панель настроек (формат, столбцы, фильтры, стили)
3. Панель управления (кнопки старт/стоп/сводная)
4. Прогресс-бар
5. Панель логов
6. Нижняя панель (открыть/удалить/отчёт/настройки)

## Бизнес-логика

### Класс `FileUtilities`
Статические методы для работы с файлами:
- `get_encoding(file_path)` - определение кодировки
- `get_delimiter(file_path)` - определение разделителя
- `is_network_path(file_path)` - проверка сетевого пути
- `sanitize_sheet_name(name)` - очистка имени листа Excel
- `count_rows(...)` - подсчёт строк с фильтром

### Класс `TSVToExcelConverter`
Поток конвертации:
- Наследуется от `QThread`
- Сигналы: `update_progress`, `log_message`, `finished_signal`, `error`
- Методы: `run()`, `stop()`, `_convert_file()`, `_init_formats()`

### Класс `PivotTableProcessor`
Обработчик сводных таблиц:
- `create_pivot_data(...)` - создание данных
- `write_pivot_table(...)` - запись в Excel

## Расширение

### Добавление нового виджета
```python
from gui import StyledButton, StylePalette

class MyCustomButton(StyledButton):
    def __init__(self, text, parent=None):
        super().__init__(text, parent, 'custom')
        # Логика
```

### Добавление новой настройки
1. Добавить элемент в `SettingsDialog._create_appearance_group()`
2. Сохранить в `get_settings()` / загрузить в `load_settings()`
3. Обработать в `main.py._show_settings()`

## Стилевые принципы

1. **Никакого QSS** - используем `QPalette`, `setFont()`, `setSizePolicy()`
2. **Только layout-ы** - никаких абсолютных координат
3. **Разделение ответственности** - GUI не знает о бизнес-логике
4. **Сигналы/слоты** - коммуникация между компонентами
5. **Поточная обработка** - долгие операции в `QThread`
