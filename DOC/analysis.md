# Анализ кода проекта TSV-Converter

## 1. Общая архитектура проекта
Проект представляет собой десктопное приложение на Python (с использованием **PySide6** для GUI), предназначенное для конвертации и обработки файлов `TSV`/`CSV` в формат `Excel (XLSX)` или `CSV`. Приложение реализует паттерн, близкий к MVC (Model-View-Controller):
- **Model / Service (Бизнес-логика):** `converter.py` (чтение, обработка данных, запись XLSX/CSV).
- **View (Интерфейс):** `gui.py` (отрисовка элементов, диалоговых окон, кастомных виджетов без QSS).
- **Controller (Связующий слой):** `main.py` (инициализация, связывание сигналов GUI с логикой обработки).

## 2. Карта файлов и модулей

### 2.1. `main.py`
Точка входа в приложение. Отвечает за инициализацию QApplication и координацию между интерфейсом (GUI) и бизнес-логикой (Конвертером).
- **Класс `TSVConverterApp`**:
  - Хранит состояние приложения и настройки (`QSettings`).
  - Создает главное окно `MainWindow`.
  - Связывает сигналы кнопок (открытие, удаление, настройки, запуск) со слотами-обработчиками (например, `_start_conversion`, `_on_files_added`).
  - Управляет фоновыми задачами (асинхронный подсчет строк: `_count_rows_task`, анализ распределения: `_count_split_distribution_task`).
  - Генерирует и сохраняет HTML-отчет о конвертации (`_export_report`).

### 2.2. `gui.py`
Файл полностью посвящен пользовательскому интерфейсу. Отличительной особенностью является использование нативной отрисовки элементов интерфейса (переопределение `paintEvent`, использование `QPalette`) с полным отказом от таблиц стилей (`QSS`), что решает проблемы совместимости и системных тем в Windows 11.
- **Основные классы и виджеты**:
  - `StylePalette`: Цветовые схемы для светлой и темной темы.
  - Кастомные кнопки и элементы: `StyledButton` (Primary, Secondary, Danger), `StyledCheckBox` (ручная отрисовка чекбокса).
  - Списки с поддержкой чекбоксов: `CheckBoxListWidget`, `CheckBoxListDelegate`.
  - Многопоточные утилиты для UI: `LoadingOverlay` (спиннер), `LoadingWorker`, `UniqueValuesWorker` (потоки `QThread` для фоновой работы без зависания GUI).
- **Окна и Диалоги**:
  - `MainWindow`: Главное окно, компановка через layout-ы (верхняя панель файлов, настройки, панель управления, прогресс-бар, лог-панель).
  - `ColumnValuesDialog`: Окно выбора значений колонки для разделения/фильтрации (асинхронно подгружает уникальные значения).
  - `PivotSettingsDialog` & `PivotPreviewDialog`: Настройки и предпросмотр сводных таблиц (Pivot Tables).
  - `SettingsDialog`: Глобальные настройки (тема, папка по умолчанию, RAM-лимиты).
  - `TSVPreviewDialog`: Предпросмотр исходного файла (пагинация, поиск с подсветкой).

### 2.3. `converter.py`
Бизнес-логика, работа с файловой системой и генерация Excel/CSV файлов. Тяжелые операции выполняются в `QThread`, отправляя сигналы о прогрессе в GUI.
- **Структуры данных (dataclasses)**: `ConversionConfig` (параметры конвертации), `ProgressData` (метрики скорости и времени).
- **Класс `FileUtilities`**: Статические утилиты для определения кодировки (по BOM и проверкам), разделителя файла, подсчета строк и валидации названий листов/файлов.
- **Класс `TSVToExcelConverter (QThread)`**:
  - Основной поток конвертации. Поддерживает запись в один файл, разделение по листам и разделение по отдельным файлам.
  - Использует `xlsxwriter`. Поддерживает оптимизацию памяти (`constant_memory: True`) при превышении порога строк (по умолчанию 500k строк).
  - Методы обработки: `_convert_to_csv`, `_convert_to_xlsx`, `_convert_with_split`, `_convert_with_split_to_files`, `_convert_without_split`.
- **Класс `PivotTableProcessor`**: Инструмент онлайн-агрегации (вычисление сводных данных "на лету" без загрузки всего файла в память). Генерирует кросс-таблицы (Строки × Столбцы = Агрегированные значения).
- **Класс `ProgressTracker`**: Подсчитывает ETA (оставшееся время), процент выполнения и скорость (строк/сек).

### 2.4. `TSV_Converter.spec`
Конфигурационный файл для сборщика `PyInstaller`.
- В `excluded_modules` исключаются тяжелые библиотеки (numpy, pandas, matplotlib, pillow), чтобы уменьшить вес итогового `.exe` файла. Это говорит о том, что вся аналитика и работа с данными написаны на чистом Python + `xlsxwriter`.

## 3. Ключевые паттерны и потоки данных (Для LLM агентов)

1. **Разделение UI и долгих операций (Threading):**
   - Никакие долгие файловые операции (подсчет строк, получение уникальных значений столбца, конвертация) не выполняются в основном потоке интерфейса (Main Thread).
   - Всегда используются `LoadingWorker`, `UniqueValuesWorker` или `TSVToExcelConverter`, которые наследуются от `QThread`.
   - Результаты передаются обратно в UI через Qt-сигналы (`Signal.emit()`).
   
2. **Управление состоянием фильтров:**
   - Состояние фильтрации и параметров разделения хранится в `main.py` в объекте `MainWindow` (в атрибутах типа `_filter_values`, `_selected_column_values`, `_pivot_settings`). Это состояние передается в `TSVToExcelConverter` (в виде `ConversionConfig`) при запуске процесса.

3. **Отрисовка UI без QSS:**
   - В проекте **категорически запрещено** использовать `widget.setStyleSheet()` для базовых контролов (особенно кнопок и чекбоксов).
   - Вместо этого используется изменение `QPalette` (см. функцию `apply_native_style` в `gui.py`) и переопределение событий отрисовки `paintEvent` (например, в `StyledCheckBox` и `CheckBoxListDelegate`).
   - Если нужно внести изменения в дизайн виджетов, необходимо менять логику внутри методов отрисовки (`paintEvent` / `paint`), а не добавлять QSS-строки.

4. **Алгоритм Конвертации (Data Flow):**
   - Сбор конфигурации: Пользователь жмет "Начать", из GUI собирается `ConversionConfig`.
   - Инициализация `TSVToExcelConverter`.
   - Предварительный подсчет общего числа строк: `_count_total_rows()` — нужно для трекера прогресса.
   - Итерация по файлам: `_convert_file()`. В зависимости от формата выбирается метод (`_convert_to_csv` / `_convert_to_xlsx`).
   - Итерация по строкам (построчное чтение для экономии памяти): Читаются строки, применяются фильтры, проверяется необходимость создания новых листов/файлов (при разделении).
   - Агрегация сводной таблицы (если включена): Выполняется отдельным проходом `PivotTableProcessor` после основной конвертации, либо данные собираются на лету (зависит от логики `PivotTableProcessor`).

5. **Оптимизация производительности:**
   - Используется инкрементальное чтение (`csv.reader`).
   - Для больших Excel-файлов включается режим `{ "constant_memory": True }` в xlsxwriter, чтобы избежать `MemoryError`.
